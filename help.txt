使用layout: false能不能实现取决于layout模板，它会将layout模板中的{{body}} 替换成你的html内容，
然后输出。虽然最终结果可能没有问题，但是实际上，它的html可能是<html><head><body> <html><body>
这种格式。并不建议这样子做，除非你把html5的<html><head><body>去掉。然后，弱弱地问一句：
难道你们不知道hexo有个skip_render: [p/**/api/*.html,p/**/api/**]
设置吗？

markdown写法

换行：

回车键。

连续3个 ― 表示分割线。

强调：

星号（*）与下划线（_）都可以，单是斜体，双是粗体，符号可跨行，符号可加空格。

有序列表：

数字 + . + 空格， 数字不能省略但可无序，点号之后的空格不能少 。

超链接：

文字：

[文字](url "可选的title")

图片：

![图片名称](图片路径)

插入代码：

新起一行，按2次Tab键或者8次空格。插入代码也可以加入有序列表。

或者

{ % codeblock [title] [lang:language] [url] [link text] % }

code snippet

{ % endcodeblock % }

防止github中的READ.me文件被渲染的方法：

在~\Hexo\source文件夹下建立READ.md文件。

在~\Hexo\ _config.yml文件中skip_render:后面加上README.md即 skip_render: README.md 。
